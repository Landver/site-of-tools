# Roadmap — client-side detection signals (G01–G25)

*(part of the [roadmap index](README.md))*

Sorted by severity then effort. Every row is one capability a competitor
provides that we don't fully match; the final column states what they do and
the recommended move for our stack. Ratings key: see
[README.md § How to read the ratings](README.md#how-to-read-the-ratings).

| # | Capability they provide | Who has it | Sev · Effort · Status | What they do that we don't → recommended move |
|---|---|---|---|---|
| G01 | Expand userAgentData high-entropy hints + platformVersion coherence | CreepJS, iphey.com, Fingerprint.com | medium · trivial · **Shipped** | Pull the full getHighEntropyValues set (architecture, bitness, model, platformVersion, uaFullVersion, fullVersionList) and cross-check against the UA. CreepJS caught a UA claiming macOS 10_15_7 while userAgentData reported macOS 26.5.1 — the frozen-Electron/spoof tell. → **We request platform ONLY. Request platformVersion + uaFullVersion + fullVersionList too and add a rule comparing UA-embedded OS version vs userAgentData.platformVersion. This is the exact Electron/spoof catch we cite in our design, made stronger for near-zero cost.** |
| G02 | navigator.productSub / oscpu / buildID / pdfViewerEnabled | iphey.com, AmIUnique.org, CreepJS | medium · trivial · **Shipped (productSub only; pdfViewerEnabled dropped as a user-setting false-fire, oscpu/buildID skipped)** | productSub is a classic engine tell (Chromium is always '20030107', Gecko '20100101'); oscpu/buildID/pdfViewerEnabled add OS/engine consistency and a headless tell (pdfViewerEnabled often false headless). → **Shipped productSub as an engine-aware consistency rule. pdfViewerEnabled was tried then dropped: it is false on ordinary desktop Chrome with "Download PDFs" set or the AlwaysOpenPdfExternally policy, so it flags a user preference, not headless.** |
| G03 | Broaden cross-context (worker/iframe/SW) comparison beyond UA | deviceandbrowserinfo.com, bot.incolumitas, CreepJS | medium · low · **Shipped** | Recompute and diff more than the UA across contexts — languages, hardwareConcurrency, platform, and even WebGL renderer/fonts — between main thread, Web Worker, Service Worker, and iframe. Caught Bright Data returning Linux in a worker while the top UA claimed Windows. → **Shipped 2026-07-17:** worker + iframe also report languages/cores/`userAgentData.platform` and the worker tries an OffscreenCanvas WebGL renderer; a Service-Worker context was added (`/botcheck-sw.js`, served by the app, no fetch handler, unregistered after the probe). Four new consistency rules (`context_language/cores/platform/webgl_mismatch`) plus the SW side of `context_ua_mismatch`, each firing only when both sides confidently report. |
| G04 | Deep native-function tamper / lie detection | CreepJS, deviceandbrowserinfo.com, bot.incolumitas, BrowserScan.net, Pixelscan, Fingerprint.com | medium · low · **Shipped** | Go well beyond a toString '[native code]' check: CreepJS's queryLies checks each API for illegal own-properties/descriptors (prototype/arguments/caller), traps whether call/new/apply/class-extends throw the correct TypeError, and detects the Function.prototype.toString Proxy that puppeteer-extra-stealth installs via error-stack frame inspection. bot.incolumitas targets stealth-plugin artefacts directly (puppeteerExtraStealthUsed, overrideTest). → **Shipped 2026-07-17:** three new probes — descriptor/own-property sanity (with per-spec enumerability: WebIDL ops are `enumerable: true`, ECMA-262 built-ins are not — asserting blanket-false false-fired every real browser), call/new `TypeError` traps, and a toString-Proxy probe (shape differential vs a control native + error-stack apply-frame inspection). Rules: `tostring_proxy` (hard), `native_descriptor_tamper` + `native_callnew_tamper` (consistency, since legit privacy extensions can patch DOM APIs). The payload is versioned (`v: 2`) so a stale cached collector skips these instead of reading as tampered. **2026-07-19 finding:** all three are evaded by the current `puppeteer-extra-plugin-stealth` — see [`../testing/findings-log.md`](../testing/findings-log.md). |
| G05 | Feature-detect true engine and compare to claimed UA _(we do a narrower version)_ | iphey.com, CreepJS | medium · low · **Shipped** | Feature-detect the actual rendering engine/version (Chromium via webkitResolveLocalFileSystemURL + BatteryManager + vendor; Gecko via buildID + onmozfullscreenchange; WebKit via ApplePayError) and cross-check against the claimed UA — catches spoofed UAs and anti-detect browsers a string parse misses. → **We compare UA vs userAgentData.platform but never feature-detect the real engine. Add a small engine-probe module and one rule (feature-detected engine family vs UA-claimed browser). Cheap, deterministic, and robust against UA spoofing.** |
| G06 | HTTP header value/presence consistency vs claimed browser | AmIUnique.org, incolumitas, DataDome | medium · low · **Shipped** | Inspect the presence and VALUES of Accept, Accept-Encoding, Upgrade-Insecure-Requests, Connection, Cache-Control etc. and check them for coherence with the claimed browser (beyond just header order). → **Shipped 2026-07-17:** three soft rules (`accept_encoding_missing`, `accept_language_missing`, `accept_nav_mismatch`) keyed on `looksLikeBrowser(UA)` — soft precisely because proxies can strip/rewrite headers (the `sec_fetch_missing` caveat). Upgrade-Insecure-Requests is captured but deliberately unused: Safari never sends it, so any rule requiring it would false-positive real Safari. |
| G07 | WebGL vendor/renderer/feature internal inconsistency | deviceandbrowserinfo.com, incolumitas | medium · low · **Shipped** | Check UNMASKED_VENDOR vs UNMASKED_RENDERER and the GPU parameter/feature set for internal self-contradiction (distinct from our software-renderer test and from GPU-vs-OS coherence). → **Shipped 2026-07-17:** the collector reports the vendor string too, and `webgl_vendor_mismatch` fires when vendor and renderer parse to different confident families (e.g. vendor Apple vs renderer NVIDIA). Verified against real reporting styles (ANGLE shim pairs, Safari's generalised "Apple Inc."/"Apple GPU", Firefox driver strings) so no real browser contradicts itself. **2026-07-19 note:** this rule silently never fired for anyone (bot or human) until a `webglGPU()` collector bug — an undefined variable — was fixed; see [`../testing/findings-log.md`](../testing/findings-log.md). |
| G08 | WebGL/GPU identity vs claimed OS/UA coherence | Pixelscan, CreepJS, DataDome, iphey.com, deviceandbrowserinfo.com | medium · low · **Shipped** | Cross-check the unmasked GPU vendor/renderer against the claimed platform (e.g. a Chrome-on-Windows UA whose canvas/WebGL renderer matches an Apple/Metal GPU is flagged). CreepJS also diffs worker-scope WebGL renderer vs main (hasBadWebGL). → **Shipped 2026-07-17:** `gpu_os_mismatch` fires only on enumerated impossible pairs (Apple GPU on Windows/Linux/Android, desktop NVIDIA/AMD on iOS/Android, Adreno/Mali on macOS/iOS) — deliberately silent on AMD+macOS (Intel Macs), Adreno+Windows (Snapdragon ARM), Intel+Android, and unknown GPUs. The worker-vs-main WebGL diff half shipped with G03 (`context_webgl_mismatch`). Same 2026-07-19 collector-bug note as G07 applies (fixed). |
| G09 | WebRTC local/public IP leak (STUN, mDNS candidates) | bot.incolumitas, CreepJS, BrowserScan.net, Pixelscan, iphey.com, whoer.net | medium · medium · **Shipped** | Use a STUN request (e.g. stun.l.google.com:19302) to enumerate ICE candidates and extract the real local (RFC1918) and public IP behind a VPN/proxy, then flag a WebRTC-real-IP vs egress-IP mismatch. → **Shipped 2026-07-18:** the collector harvests candidate IPs over a public STUN server (~1.5 s, mDNS `.local` names skipped); `webrtc_ip_mismatch` (consistency) fires only when a PUBLIC candidate differs from the server-observed egress IP — private/loopback/link-local/ULA/CGNAT candidates excluded (normal NAT, never a tell), same address family only (dual-stack stays silent), empty candidates/egress ⇒ skip. |
| G10 | Cheap headless render tells (battery, broken-image, hairline, system-color) | sannysoft, CreepJS, bot.incolumitas, iphey.com | low · trivial · **Shipped (broken-image only)** | Small client render/behavior probes: getBattery presence/behavior (CHR_BATTERY), broken-image 0x0 natural dimensions, Modernizr 0.5px hairline offsetHeight quirk, and CreepJS's hasKnownBgColor (render CSS system color ActiveText, expect rgb(255,0,0) in headless). → **Shipped 2026-07-18 (broken-image only):** `image_broken` (soft) — a 1×1 data-URI image that must load in any real browser; naturalWidth == 0 or an error event ⇒ stripped/headless. hasKnownBgColor was built then **dropped before shipping**: ground-truthed in a real, headed Chrome 150 on macOS, where ActiveText computes to exactly rgb(255,0,0) — the "headless default" is now every real Chrome, so the tell is dated. Battery/hairline skipped as dated. |
| G11 | iframe proxy/override detection + webdriver-in-iframe | CreepJS, deviceandbrowserinfo.com | low · trivial · **Shipped** | CreepJS's hasIframeProxy builds a srcdoc iframe and inspects contentWindow for a proxied window; deviceandbrowserinfo re-checks navigator.webdriver inside an iframe (hasWebdriverInFrameTrue) to catch evasions that only patch the main frame. → **Shipped 2026-07-18:** `iframe_webdriver` (hard — the iframe's fresh realm has its own Navigator.prototype and leaks what the patched top frame hides) and `iframe_proxy` (consistency — the stealth contentWindow getter verifiably throws when a fresh srcdoc frame's window is read, CreepJS's exact hasIframeProxy mechanics). |
| G12 | Additional fingerprint-entropy surfaces (audio, WebGPU, DOMRect, media devices, speech, touch) | CreepJS, bot.incolumitas, Fingerprint.com, BrowserScan.net, iphey.com, AmIUnique.org, EFF Cover Your Tracks | low · low · **Shipped (touch only; rest deferred)** | Collect AudioContext fingerprint, WebGPU capabilities, DOMRect/SVGRect/clientRects geometry, media-device enumeration, speech-synthesis voices, and touch/maxTouchPoints as fingerprint entropy and (in CreepJS) cross-context consistency inputs. → **Shipped 2026-07-18 (touch only):** `mobile_no_touch` (consistency, v3-gated) — an Android/iOS UA reporting maxTouchPoints == 0, which no real phone does. The reverse direction (desktop UA + touch) is deliberately not a rule: touch-screen Windows laptops would false-fire constantly. Audio and the remaining entropy surfaces stay deferred (value only with a crowd/rarity DB). |
| G13 | Broaden automation-framework signature battery | BrowserScan.net, sannysoft, deviceandbrowserinfo.com, bot.incolumitas | low · low · **Shipped** | Cover more frameworks by name: Sequentum (window.external), Awesomium, CEF/CefSharp, FMiner, Rhino, WebdriverIO, SlimerJS, plus puppeteer-extra-stealth and puppeteer evaluation-script artefacts. BrowserScan reports 15+ individually. → **Shipped 2026-07-18:** WINDOW_MARKERS extended with the Playwright binding hooks (`__pwInitScripts` / `__playwright__binding__`), the wider Selenium/Watir canon (the intoli/fp-scanner lineage sannysoft runs), and Sequentum via `window.external`. CefSharp/Awesomium/CEF deliberately excluded from the hard-60 list — legit desktop apps embed those runtimes, and the UA-side `embedded_runtime` consistency rule already covers that class. **2026-07-19 finding:** the Playwright binding-hook markers don't fire against a stock `playwright` launch with no bindings exposed — see [`../testing/findings-log.md`](../testing/findings-log.md). |
| G14 | CDP / navigator recheck in a Service Worker context | bot.incolumitas, CreepJS | low · low · **Shipped, downgraded 2026-07-19** | Run CDP and navigator consistency checks inside a Service Worker (not just a Web Worker); bot.incolumitas's inconsistentServiceWorkerNavigatorPropery flagged an Electron/CDP browser whose main-thread webdriver read clean. → **Shipped 2026-07-18:** `/botcheck-sw.js` also reports `navigator.webdriver` and runs the same CDP Error.stack trap the worker probe uses. Rules: `webdriver_sw` (hard) and `cdp_sw_only` (soft as of 2026-07-19, was consistency). **2026-07-19 finding:** confirmed across 3 real automation frameworks that `webdriver_sw` never reads true even when webdriver is true elsewhere in the same session, and the CDP trap never fires at all — see [`../testing/findings-log.md`](../testing/findings-log.md) for the full audit. |
| G15 | CSS media-query / computed-style / display-capability fingerprint | CreepJS, iphey.com | low · low · **Shipped (media-query probes only)** | Probe prefers-color-scheme, color gamut, HDR/HDCP, forced/inverted colors, reduced-motion/transparency, devicePixelRatio, CSS system colors and computed styles. → **Cheap client probes; add a few (devicePixelRatio vs screen, forced-colors) as consistency/entropy surfaces.** → **Shipped 2026-07-18 (wave-2 probes batch):** the collector payload is now `v: 4` with one additive `env` section — devicePixelRatio, prefers-color-scheme, forced-colors, reduced-motion, dynamic-range/gamut (all fail-to-absent, all entropy-only: user preferences and hardware capabilities are never scored) — plus one rule: `matchmedia_missing` (soft, v4-gated): a browser UA without `window.matchMedia` is a stripped JS environment. The DPR-vs-screen consistency rule was **deliberately not built**: zoom legitimately changes DPR and the inner-window sizes while `screen.*` stays zoom-invariant in Chrome, so zoomed-out real windows false-fire it; CSS system colors stay dropped (real Chrome computes ActiveText to the old headless value). |
| G16 | DevTools-open detection (debugger timing + window-size delta) | iphey.com, Fingerprint.com | low · low · **Not built — re-evaluate priority** | iphey times a `debugger` statement in a worker and checks outerWidth - innerWidth > 160px; Fingerprint exposes a 'Developer Tools' Smart Signal. A weaker automation proxy than CDP but a distinct signal. → **Originally skipped as "redundant with the CDP trap" — that assumption is now wrong (see G14/CDP-trap finding above), so this is worth re-evaluating. Caveat: it detects a human with DevTools open, not automation — Puppeteer/Playwright/Selenium don't open a visible DevTools panel by default, so this wouldn't have caught anything in the 2026-07-19 audit either. Re-scope what it's actually good for before building it.** |
| G17 | Full window-global enumeration + navigator prototype descriptor walk | sannysoft, CreepJS, iphey.com, Pixelscan | low · low · **Shipped** | Enumerate all window globals and walk navigator's prototype property descriptors to surface injected automation globals and getter overrides (faked webdriver/plugins), rather than only reading spoofable values. → **Shipped 2026-07-18:** `navigator_proto_tamper` (consistency, v3-gated) — per WebIDL, `webdriver`/`plugins`/`languages` must be native, getter-only, enumerable+configurable accessors living on Navigator.prototype, never own data properties on the instance; only confident anomalies fire, probe failure ⇒ pass. The suspect-name own-property sweep now covers `window` as well as `document`, folded into the existing `framework_globals`. **2026-07-19 finding:** evaded by the current `puppeteer-extra-plugin-stealth` — see [`../testing/findings-log.md`](../testing/findings-log.md). |
| G18 | Impossible-geometry / recursion-overflow headless tells | bot.incolumitas, sannysoft | low · low · **Not built** | overflowTest / resOverflow deliberately trigger a recursion stack overflow and read the error message/signature; plus impossible screen/window geometry checks used as headless indicators. → **Mostly legacy PhantomJS-era tells with limited modern value. | G19 | Incognito / private-mode detection | Fingerprint.com, BrowserScan.net, Pixelscan | low · low · **Not built** | Detect private/incognito mode via storage-quota and filesystem-API heuristics and surface it as a signal. → **Low value as a bot tell (humans use incognito constantly) and the detection tricks are brittle/version-specific. | G20 | Privacy/anti-detect tool resistance detection | CreepJS | low · low · **Not built** | Detect Tor Browser, Firefox RFP, Brave, ungoogled-chromium, and extensions (uBlock/NoScript/CanvasBlocker/Chameleon/ScriptSafe) and measure how well the mask holds. → **Niche (serves the anti-detect audience, not bot detection). Skip, or add a small informational readout later. Not a scoring signal for us.** |
| G21 | Storage/quota, Network Information, MediaCapabilities/EME, GPC, full Permissions enumeration | iphey.com, CreepJS, AmIUnique.org, incolumitas | low · low · **Shipped (as entropy + one coherence rule)** | Probe localStorage/indexedDB presence + quota, Network Information (rtt/downlink/effectiveType — incolumitas' connectionRTT), MediaCapabilities/EME-DRM, Global Privacy Control, and enumerate all Permissions states. → **Cheap additional entropy/consistency surfaces; connectionRTT vs IP geo is a genuinely new cross-check.** → **Shipped 2026-07-18 (wave-2 probes batch, in the v4 `env` section):** a `navigator.connection` sample (effectiveType/downlink/rtt/saveData — absent on Firefox/Safari, which is normal), rounded storage quota MB, GPC, a two-name Permissions sample, and EME ClearKey availability — all entropy for the raw dump, never scored (preferences and capabilities aren't bot tells). One rule: `netinfo_incoherent` (soft, v4-gated) — effectiveType claiming faster than its own reported rtt/downlink (the browser derives the type from exactly those numbers; thresholds graced by the API's rounding). **Deliberately skipped:** incognito detection via quota (that's G19), the rtt-vs-IP-geo cross-check (the client rtt measures the same egress path the geo describes, so ordinary VPN users would false-fire), full Permissions enumeration (a two-name sample carries the entropy at no cost), and MediaCapabilities beyond ClearKey EME. |
| G22 | chrome.runtime integrity + late-injection index checks | CreepJS | low · low · **Shipped** | hasBadChromeRuntime instantiates chrome.runtime.sendMessage/connect and inspects for missing prototype / wrong error constructor to unmask a faked chrome object; hasHighChromeIndex flags 'chrome' appearing among the last ~50 window keys (stealth patches inject it late). → **Shipped 2026-07-18:** `chrome_runtime_tamper` (consistency, v3-gated — genuine runtime `sendMessage`/`connect` are native non-constructors: no own `prototype`, `new fn()` throws a `TypeError`; only confident contradictions fire) and `chrome_late_injection` (consistency — 'chrome' in the last ~50 of both the enumerable keys and the own property names). Both gated on a Chrome UA. **2026-07-19 finding:** both evaded by the current `puppeteer-extra-plugin-stealth`. A fix for `chrome_runtime_tamper` (flag `window.chrome` existing without `runtime` at all) was written and verified against stealth, then **reverted** — the official "Chrome for Testing" binary itself lacks `chrome.runtime` regardless of automation, and there was no genuine consumer-Chrome sample available to rule out false-positiving real users. See [`../testing/findings-log.md`](../testing/findings-log.md) — this is flagged there as the single most valuable open item. |
